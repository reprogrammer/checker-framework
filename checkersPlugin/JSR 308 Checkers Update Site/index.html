What it is:
Pluggable type-checkers are bug-finding and verification tools for preventing and detecting null pointer errors, incorrect uses of equality operations, and side effect errors.
The checkers demonstrate the usefulness of the type annotations.

Requirements:
For the plugin to work, your Eclipse must be running on a Java 6 VM or newer.
The plugin has been tested on Linux, Eclipse 3.3.2 and Java 6.
To see what Java VM and Eclipse is running on, select from the menu Help -> About Eclipse Platform -> Configuration Details.
Depending on your installation, the menu may be Help -> About Eclipse SDK -> Configuration Details.

How to install:
  1. In Eclipse, select from the menu Help -> Software Update -> Find and Install...
  2. Choose the Search for new features to install option, and click Next.
  3. Click New Remote Site.
  4. Enter the following:
    • Name:JSR 308 Checkers
    • URL:http://types.cs.washington.edu/jsr308/eclipse
    and click OK.
  5. Under Sites to include in search, you should see "JSR 308 Checkers".
    Select the checkbox next to it and click Finish.
  6. Under Select features to install, you should see "JSR 308 Checkers Feature".
    (You may have to expand the tree to make the feature visible.)
    Select the checkbox next to it and click Next.
  7. Select the I accept the terms in the licence agreement option and click Next.
  8. Select the location. The default (your workspace) should be fine. Click Finish.
  9. The plugin is not digitally signed. Install it anyway.
  10. Click Yes to restart Eclipse.
How to use Checkers Feature:
First, you need to add checkers.jar to your project's classpath.
(Eclipse may complain about unresolved classes referenced form a classfile. This does not affect the checking but if you want to make the warning go away, add javac.jar to the project's classpath.)
To perform the checking, select the checker from the context menu of a Java project, for example JSR 308 -> Run Nullness.
The checker will run and warnings will be added to the Problems view.
To remove the warning markers, run JSR 308 -> Remove Markers

How to use the Skeleton Generator Feature:
  1. Create a new Eclipse project containing the desired packages. The plug-in performs in-place modifications to the files. They need to be in the source path (usually in src directory), but do not need to have the classpath configured properly.
  2. Select (i.e. highlight) the packages (or individual source files) to be converted.
  3. Choose JSR308 Tools > Skeleton Files (With JavaDocs) or JSR308 Tools > Skeleton Files (No JavaDocs) from their context menus. The documentation might be helpful when annotating the code.
  4. The plug-in generates skeleton files in place of the input files.
How to report problems:
Submit bug reports at http://code.google.com/p/jsr308-langtools/issues/list

Currently supported checkers: NonNull, Javari, IGJ, Interned
